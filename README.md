# Totocalcio 2026 — World Cup 予測最適化プロジェクト

家族でやる「お金をかけない」トトカルチョ（2026 FIFAワールドカップ）の
**配点を数理最適化**するためのquantプロジェクト。

## ゲームのルール

- 参加者はそれぞれ **10カ国** を選ぶ。
- 「予選リーグ＋決勝トーナメントを通して**勝ち試合が多くなりそう**な国」から順に
  **10点 → 9点 → … → 1点** を配分する（合計55点ぶんの予想）。
- ある国が**試合に勝つたび**、その国に点を付けた人へ **他の参加者全員がその点を支払う**
  （**ゼロサム精算**）。
- **「勝ち」の定義**：
  - グループステージの**引き分けはノーカウント**。
  - 決勝トーナメントで **PK戦で勝った場合も「1勝」とみなす**。

## 核心の数学：このゲームは「期待勝利数ランキング」に帰着する

記号：
- `vₚ(C)` … 参加者 `P` が国 `C` に付けた点（選んだ10カ国は10〜1、未選択は0）
- `S_C` … 全員がその国に付けた点の合計
- `N` … 参加人数
- `W_C` … 大会を通じてその国が挙げた勝利数（PK勝ち込み・グループ引分け除外）

1勝あたりの `P` の純収支：`N·vₚ(C) − S_C`。大会全体では

```
P の総得点 = Σ_C W_C · (N·vₚ(C) − S_C)
```

特定の相手 `Q` との差は共通項が消えて

```
P − Q = N · Σ_C W_C · (vₚ(C) − v_Q(C))
```

→ 期待値では、相手が誰でも **`Σ_C E[W_C]·vₚ(C)` を最大化**すればよい。
重み `{10,9,…,1}` は固定なので、**並べ替え不等式より
「E[W_C] が大きい国に大きい点を振る」のが厳密に最適**。

### したがってプロジェクトの本丸は一つ
**各国の期待勝利数 `E[W_C]` をどれだけ正確に推定できるか。**
Elo・ポアソン・モンテカルロ・ブックメーカーオッズは全てこの推定を磨く道具。

### 第2レイヤー：1位確率の最適化（任意）
N人ゼロサムでは全員が本命に高得点を集中させ差がつきにくい。
「家族内で1位になる確率」を上げるには、EVを少し犠牲にしてでも
**人と違う高分散ピック**を混ぜる価値がある（DFS/ポーカーのGPP最適化と同型）。
→ 対戦相手モデル＋標準偏差を考慮した配点最適化で扱う。

## プロジェクト構成

```
totocalcio/
├── data/        # 生データ: 組分け, Elo, FIFAランク, ブックオッズ, 過去試合結果
├── ratings/     # チーム強度モデル (Elo / Poisson Dixon-Coles / オッズ統合)
├── sim/         # トーナメント・モンテカルロ (2026ブラケットを忠実に再現)
├── optimize/    # 配点最適化: ①EV最適 ②1位確率最適(対戦相手モデル+分散)
├── report/      # 各国 E[W]・分布・推奨配点の可視化
└── README.md
```

## 開発フェーズ

| # | フェーズ | 内容 |
|---|---|---|
| 0 | データ取得 | 確定した2026組分け・日程・各種レーティング・オッズを収集 |
| 1 | 強度モデル | レーティング→試合結果確率（PK込み）。市場オッズで較正 |
| 2 | シミュレーション | 48カ国・12組・R32〜決勝を10万回試行 → E[W_C] と分布 |
| 3 | 配点最適化 | EV最適ランキング ＋ 1位確率最適（対戦相手モデル） |
| 4 | レポート | 推奨10カ国と配点、感度分析の可視化 |
